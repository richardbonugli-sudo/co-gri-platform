1. Overview & Architecture

1.1 System Architecture

Step 1 processes geographic exposure evidence for four independent channels with the following blended weights:

Channel Symbol Default Weight Description
Revenue Wᵣ 40% Geographic revenue distribution
Supply Chain Wₛ 35% Manufacturing, sourcing, suppliers
Physical Assets Wₚ 15% PP&E, facilities, subsidiaries
Financial W𝒻 10% Debt, currency, treasury centers
📌 Key Principle: Each channel is calculated independently, then blended using sector-specific coefficients. This ensures that evidence quality in one channel does not contaminate others.

1.2 Evidence Hierarchy (4 Tiers)

The system follows a strict 4-tier evidence hierarchy:

Tier Evidence Type Confidence Priority
Tier 1 Structured Primary Evidence (Tables, Exhibit 21) 90-95% Highest
Tier 2 Narrative Primary Evidence (SEC text) 70-85% High
Tier 3 Supplementary Evidence (Sustainability reports) 60-80% Medium
Tier 4 Fallback Logic (SSF, RF, GF) 50-75% Last Resort

2. Structured Evidence Parsing

2.1 Table Detection & Parsing Algorithms

2.1.1 Revenue Table Detection

The system searches for geographic revenue tables in SEC 10-K/10-Q filings using pattern matching:

FUNCTION detectRevenueTable(secFilingText):
    // Step 1: Identify table sections
    tableSections = extractTableSections(secFilingText)
    
    FOR EACH section IN tableSections:
        // Step 2: Check for geographic revenue indicators
        hasGeographicHeader = containsKeywords(section.headers, [
            "geographic", "region", "segment", "revenue by", 
            "net sales", "revenues"
        ])
        
        hasRevenueData = containsKeywords(section.content, [
            "revenue", "sales", "net sales"
        ])
        
        IF hasGeographicHeader AND hasRevenueData:
            // Step 3: Parse table structure
            parsedTable = parseTableStructure(section)
            
            // Step 4: Extract country/region data
            RETURN extractGeographicData(parsedTable)
    
    RETURN NULL // No revenue table found

2.1.2 Geographic Entity Recognition

Once a table is detected, the system extracts geographic entities (countries and regions):

FUNCTION extractGeographicData(parsedTable):
    geographicData = {
        countries: [],
        regions: [],
        weights: {},
        totalAllocated: 0.0
    }
    
    FOR EACH row IN parsedTable.rows:
        entityName = row.geographicEntity
        revenueValue = row.revenueAmount
        
        // Step 1: Normalize entity name
        normalizedName = normalizeEntityName(entityName)
        
        // Step 2: Classify as country or region
        IF isActualCountry(normalizedName):
            countryName = normalizeCountryName(normalizedName)
            geographicData.countries.append(countryName)
            geographicData.weights[countryName] = revenueValue
        ELSE IF isKnownRegion(normalizedName):
            geographicData.regions.append(normalizedName)
            geographicData.weights[normalizedName] = revenueValue
        
        geographicData.totalAllocated += revenueValue
    
    RETURN geographicData

2.1.3 Revenue/Percentage Extraction

The system extracts numerical values and converts them to normalized weights:

FUNCTION parsePercentage(valueString):
    // Handle various formats: "45.2%", "$169,658M", "45.2"
    
    // Step 1: Remove currency symbols and commas
    cleanValue = valueString.replace(/[$,]/g, "")
    
    // Step 2: Check if percentage or absolute value
    IF cleanValue.contains("%"):
        percentage = parseFloat(cleanValue.replace("%", ""))
        RETURN percentage / 100.0
    ELSE:
        // Absolute value - will be normalized later
        RETURN parseFloat(cleanValue)

FUNCTION normalizeWeights(weights):
    // Ensure all weights sum to 1.0
    totalWeight = sum(weights.values())
    
    normalizedWeights = {}
    FOR EACH entity, weight IN weights:
        normalizedWeights[entity] = weight / totalWeight
    
    RETURN normalizedWeights

2.2 Data Validation Rules

2.2.1 Validation Rule Set

All structured evidence must pass the following validation rules:

Rule ID Rule Name Description Severity
V1 Sum of Weights = 1 Total allocation must equal 100% (±1% tolerance) ERROR
V2 No Country in Multiple Regions Each country appears in exactly one region ERROR
V3 Non-Negative Weights All weights must be ≥ 0 ERROR
V4 Valid Geographic Entities All entities must be recognized countries or regions WARNING
V5 Minimum Threshold Weights below 0.1% may be excluded INFO

2.2.2 Validation Implementation

FUNCTION validateStructuredEvidence(geographicData):
    validationResults = []
    
    // Rule V1: Sum of Weights = 1
    totalWeight = sum(geographicData.weights.values())
    sumValid = abs(totalWeight - 1.0) < 0.01
    
    validationResults.append({
        rule: "V1: Sum of Weights = 1",
        passed: sumValid,
        message: "Total weight: " + formatPercent(totalWeight),
        severity: sumValid ? "INFO" : "ERROR"
    })
    
    // Rule V2: No Country in Multiple Regions
    countryAppearances = countCountryAppearances(geographicData)
    multipleAppearances = 0
    
    FOR EACH country, count IN countryAppearances:
        IF count > 1:
            multipleAppearances++
            validationResults.append({
                rule: "V2: No Country in Multiple Regions",
                passed: FALSE,
                message: country + " appears in " + count + " regions",
                severity: "ERROR"
            })
    
    IF multipleAppearances == 0:
        validationResults.append({
            rule: "V2: No Country in Multiple Regions",
            passed: TRUE,
            message: "All countries appear in exactly one region",
            severity: "INFO"
        })
    
    // Rule V3: Non-Negative Weights
    FOR EACH entity, weight IN geographicData.weights:
        IF weight < 0:
            validationResults.append({
                rule: "V3: Non-Negative Weights",
                passed: FALSE,
                message: entity + " has negative weight: " + weight,
                severity: "ERROR"
            })
    
    RETURN validationResults

2.3 Confidence Scoring

2.3.1 Confidence Score Calculation

Structured evidence confidence is calculated based on multiple factors:

Confidence Score Formula:

C = Cbase + Ccompleteness + Cvalidation + Ccross-check
Component Range Description
Cbase 0.85 - 0.90 Base confidence for structured tables
Ccompleteness 0.00 - 0.05 Bonus for data completeness (all fields present)
Cvalidation 0.00 - 0.03 Bonus for passing all validation rules
Ccross-check 0.00 - 0.02 Bonus for cross-validation with other sources 
FUNCTION calculateStructuredConfidence(geographicData, validationResults):
    // Base confidence for structured data
    baseConfidence = 0.90
    
    // Completeness bonus (0-5%)
    totalEntities = geographicData.countries.length + geographicData.regions.length
    completenessBonus = min(0.05, totalEntities / 10 * 0.05)
    
    // Validation bonus (0-3%)
    passedValidations = count(validationResults, v => v.passed)
    totalValidations = validationResults.length
    validationBonus = (passedValidations / totalValidations) * 0.03
    
    // Cross-validation bonus (0-2%)
    crossCheckBonus = geographicData.crossValidated ? 0.02 : 0.0
    
    confidence = baseConfidence + completenessBonus + validationBonus + crossCheckBonus
    
    RETURN min(0.95, confidence) // Cap at 95%

3.1 Narrative Parsing Algorithms

3.1.1 Text Extraction from SEC Filings

The system extracts narrative text from specific SEC filing sections:

FUNCTION extractNarrativeContent(secFiling, channel):
    // Define section mappings for each channel
    sectionMappings = {
        "revenue": ["Item 1. Business", "Geographic Segments", "Revenue Recognition"],
        "supply": ["Item 1. Business", "Manufacturing", "Supply Chain"],
        "assets": ["Item 2. Properties", "Facilities"],
        "financial": ["Item 7A. Quantitative and Qualitative Disclosures", "Note 4"]
    }
    
    relevantSections = sectionMappings[channel]
    narrativeText = ""
    
    FOR EACH sectionName IN relevantSections:
        section = secFiling.findSection(sectionName)
        IF section EXISTS:
            narrativeText += section.text + "\n\n"
    
    RETURN narrativeText

3.1.2 Geographic Entity Extraction (NLP)

Natural Language Processing is used to identify geographic entities in narrative text:

FUNCTION extractGeographicEntities(narrativeText):
    entities = {
        countries: new Set(),
        regions: [],
        descriptions: []
    }
    
    // Step 1: Sentence tokenization
    sentences = narrativeText.split(/[.;]/)
    
    FOR EACH sentence IN sentences:
        // Step 2: Country name matching
        FOR EACH country, aliases IN COUNTRY_ALIASES:
            FOR EACH alias IN aliases:
                regex = new RegExp("\\b" + alias + "\\b", "i")
                IF regex.test(sentence):
                    entities.countries.add(country)
                    BREAK
        
        // Step 3: Regional definition extraction
        regionalDef = parseRegionalDefinition(sentence)
        IF regionalDef:
            entities.regions.append(regionalDef)
        
        // Step 4: Store context
        IF entities.countries.size > 0 OR regionalDef:
            entities.descriptions.append(sentence.trim())
    
    RETURN entities

3.1.3 Regional Definition Parsing

The system identifies explicit regional definitions using pattern matching:

FUNCTION parseRegionalDefinition(sentence):
    // Pattern 1: "Region includes X, Y, and Z"
    includesPattern = /([A-Z][a-z\s]+)\s+includes?\s+([^.;]+)/i
    match = sentence.match(includesPattern)
    
    IF match:
        regionName = match[1].trim()
        includedText = match[2]
        countries = extractCountriesFromText(includedText)
        
        IF countries.length > 0:
            RETURN {
                regionName: regionName,
                countries: countries,
                source: "Narrative text (includes pattern)",
                confidence: "high"
            }
    
    // Pattern 2: "Region consists of X, Y, and Z"
    consistsPattern = /([A-Z][a-z\s]+)\s+consists?\s+of\s+([^.;]+)/i
    match = sentence.match(consistsPattern)
    
    IF match:
        // Similar processing...
        RETURN regionalDefinition
    
    RETURN NULL

3.2 Label Classification System

3.2.1 Label Types & Definitions

The system classifies narrative evidence into three label types:

Label Type Definition Confidence Usage
Direct Explicit country mention with quantitative context 85-95% Primary allocation
Indirect Country mentioned without quantitative data 60-75% Universe expansion only
Qualitative Regional description without specific countries 50-65% Restricted fallback guidance

3.2.2 Label Classification Algorithm

FUNCTION classifyNarrativeLabel(sentence, country, channel):
    // Define channel-specific keywords
    channelKeywords = {
        "revenue": {
            direct: ["revenue", "sales", "$", "%", "million", "billion"],
            indirect: ["market", "customers", "demand"]
        },
        "supply": {
            direct: ["manufacturing", "supplier", "production", "assembly"],
            indirect: ["sourcing", "procurement"]
        },
        // ... other channels
    }
    
    lowerSentence = sentence.toLowerCase()
    
    // Check for direct evidence (quantitative)
    hasQuantitative = containsAny(lowerSentence, ["$", "%", "million", "billion"])
    hasDirectKeyword = containsAny(lowerSentence, channelKeywords[channel].direct)
    
    IF hasQuantitative AND hasDirectKeyword:
        RETURN {
            label: "direct",
            confidence: 0.90,
            context: sentence,
            reasoning: "Quantitative data with direct channel keyword"
        }
    
    // Check for indirect evidence (qualitative)
    hasIndirectKeyword = containsAny(lowerSentence, channelKeywords[channel].indirect)
    
    IF hasIndirectKeyword:
        RETURN {
            label: "indirect",
            confidence: 0.70,
            context: sentence,
            reasoning: "Qualitative mention with indirect channel keyword"
        }
    
    // Default: qualitative only
    RETURN {
        label: "qualitative",
        confidence: 0.55,
        context: sentence,
        reasoning: "Country mentioned without channel-specific context"
    }

3.2.3 Label Usage Rules

📌 Critical Rule: Narrative evidence with indirect or qualitative labels is used ONLY for universe expansion (defining the set of plausible countries), NOT for direct weight allocation. 
FUNCTION applyNarrativeEvidence(narrativeLabels, channel):
    directCountries = {}
    universeExpansion = new Set()
    
    FOR EACH label IN narrativeLabels:
        IF label.type == "direct":
            // Direct labels can be used for allocation
            directCountries[label.country] = label.weight
        ELSE IF label.type IN ["indirect", "qualitative"]:
            // Indirect/qualitative only expand universe
            universeExpansion.add(label.country)
    
    RETURN {
        directAllocation: directCountries,
        expandedUniverse: universeExpansion
    }

3.3.2 Ambiguity Scoring

Narrative evidence is scored for ambiguity to determine confidence:

FUNCTION calculateAmbiguityScore(narrativeEvidence):
    ambiguityFactors = {
        multipleInterpretations: 0.0,
        vagueTerritories: 0.0,
        conflictingStatements: 0.0
    }
    
    // Factor 1: Multiple possible interpretations
    IF narrativeEvidence.contains("other") OR 
       narrativeEvidence.contains("various") OR
       narrativeEvidence.contains("certain"):
        ambiguityFactors.multipleInterpretations = 0.3
    
    // Factor 2: Vague territorial descriptions
    vagueTerms = ["region", "area", "territory", "zone"]
    IF containsAny(narrativeEvidence, vagueTerms):
        ambiguityFactors.vagueTerritories = 0.2
    
    // Factor 3: Conflicting statements in same document
    IF detectConflicts(narrativeEvidence):
        ambiguityFactors.conflictingStatements = 0.4
    
    totalAmbiguity = sum(ambiguityFactors.values())
    
    RETURN {
        score: totalAmbiguity,
        level: totalAmbiguity > 0.5 ? "high" : 
               totalAmbiguity > 0.2 ? "medium" : "low",
        factors: ambiguityFactors
    }

4. Closed-Total / SSF Gating Logic

4.1 Closed-Total Definition & Criteria

4.1.1 What is a "Closed-Total"?

Definition: A segment is considered "closed-total" when:
  1. The region membership is fully known (all countries in the region are identified)
  2. The region total is explicitly stated in structured evidence
  3. The region definition is unambiguous (no conflicting interpretations)

4.1.2 Closed-Total Detection Algorithm

FUNCTION isClosedTotal(segment, structuredEvidence, narrativeEvidence):
    // Criterion 1: Region membership fully known
    regionName = segment.name
    
    // Check if region is in UN M49 standard definitions
    IF regionName IN UN_M49_REGIONS:
        regionCountries = UN_M49_REGIONS[regionName]
        membershipKnown = TRUE
    // Check if narrative provides explicit definition
    ELSE IF narrativeEvidence.hasDefinition(regionName):
        regionalDef = narrativeEvidence.getDefinition(regionName)
        regionCountries = regionalDef.countries
        membershipKnown = regionalDef.confidence == "high"
    ELSE:
        membershipKnown = FALSE
    
    // Criterion 2: Region total explicitly stated
    totalStated = structuredEvidence.hasExplicitTotal(regionName)
    
    // Criterion 3: Unambiguous definition
    ambiguityScore = calculateAmbiguityScore(narrativeEvidence)
    unambiguous = ambiguityScore.level == "low"
    
    // All three criteria must be met
    isClosedTotal = membershipKnown AND totalStated AND unambiguous
    
    RETURN {
        isClosedTotal: isClosedTotal,
        membershipKnown: membershipKnown,
        totalStated: totalStated,
        unambiguous: unambiguous,
        regionCountries: regionCountries
    }

Open-Total Example (Apple - Rest of Asia Pacific):

  - Unambiguous: NO ("other" creates ambiguity)
  
→ Result: SSF required to distribute 3.8% across plausible countries

4.2 SSF Gating Conditions & Thresholds

4.2.1 When SSF is Triggered

Segment-Specific Fallback (SSF) is triggered when:

Condition Description Action
Region Defined Region membership is fully known (closed-total) Apply SSF within region only
Partial Evidence Some countries explicitly named, others implicit Apply SSF to residual portion
Known Zero Certain countries explicitly excluded Exclude from SSF universe

4.2.2 SSF Gating Decision Logic

FUNCTION shouldApplySSF(segment, evidence):
    // Gate 1: Check if region membership is known
    closedTotalCheck = isClosedTotal(segment, evidence.structured, evidence.narrative)
    
    IF closedTotalCheck.isClosedTotal:
        // Gate 2: Check if there's a residual portion
        explicitCountries = evidence.structured.getExplicitCountries(segment)
        totalExplicit = sum(explicitCountries.values())
        residualPortion = segment.total - totalExplicit
        
        IF residualPortion > 0.01: // >1% residual
            RETURN {
                applySSF: TRUE,
                reason: "Closed-total with residual portion",
                ssfUniverse: closedTotalCheck.regionCountries,
                residualWeight: residualPortion
            }
        ELSE:
            RETURN {
                applySSF: FALSE,
                reason: "Closed-total fully allocated"
            }
    ELSE:
        // Not closed-total → use RF or GF instead
        RETURN {
            applySSF: FALSE,
            reason: "Region membership unknown, use RF/GF"
        }

4.2.3 SSF Formula

When SSF is triggered, weights are distributed using industry demand proxies:

SSF Weight Formula:

WSSF(c) = [IndustryDemandProxy(c) / Σ IndustryDemandProxy(c)] × Wresidual

where c ∈ R (countries in defined region R) 
FUNCTION applySSF(regionCountries, residualWeight, sector, channel):
    ssfWeights = {}
    totalDemandProxy = 0.0
    
    // Step 1: Calculate industry demand proxy for each country
    FOR EACH country IN regionCountries:
        demandProxy = getIndustryDemandProxy(country, sector, channel)
        ssfWeights[country] = demandProxy
        totalDemandProxy += demandProxy
    
    // Step 2: Normalize and scale by residual weight
    FOR EACH country, demandProxy IN ssfWeights:
        normalizedWeight = (demandProxy / totalDemandProxy) * residualWeight
        ssfWeights[country] = normalizedWeight
    
    RETURN ssfWeights

5. RF Case Selection (RF-A / RF-B / RF-C / RF-D)

5.1 RF Case Definitions & Criteria

5.1.1 Overview of RF Cases

Restricted Fallback (RF) is applied when region membership is partially known or ambiguous. There are four distinct RF cases:

RF Case Trigger Condition Confidence Universe Construction
RF-A Non-standard region naming (e.g., "EMEA", "International") 60-70% Plausible countries based on sector + home country
RF-B Partial country evidence + incomplete membership 65-75% Explicit countries + similar countries
RF-C Domestic + ambiguous foreign bucket 60-70% All countries except home country
RF-D Multiple ambiguous segments 55-65% Union of all plausible countries

5.1.2 RF-A: Non-Standard Region Naming

RF-A Trigger: Segment name uses non-standard regional terminology that doesn't match UN M49 definitions. 
FUNCTION isRF_A_Case(segmentName):
    // Non-standard region keywords
    nonStandardRegions = [
        "EMEA", "APAC", "LATAM", "International", 
        "Rest of World", "Other", "Foreign", "Overseas"
    ]
    
    normalizedName = segmentName.toLowerCase().trim()
    
    FOR EACH keyword IN nonStandardRegions:
        IF normalizedName.contains(keyword.toLowerCase()):
            RETURN TRUE
    
    RETURN FALSE

RF-A Example: "International" Segment

Segment: "International"
Revenue: $50M (25%)

Analysis:
- "International" is non-standard (not in UN M49)
- No explicit country list provided
- Could mean any non-domestic country

RF-A Universe Construction:
1. Exclude home country (e.g., United States)
2. Include top 30 countries by sector relevance
3. Apply GDP × SectorPrior weighting

Result: 25% distributed across ~30 plausible countries

5.1.3 RF-B: Partial Country Evidence

RF-B Trigger: Some countries are explicitly named, but the list is clearly incomplete (e.g., "China, Japan, and other Asian countries"). 
FUNCTION isRF_B_Case(segment, evidence):
    explicitCountries = evidence.getExplicitCountries(segment)
    
    // Check for "other" or "additional" keywords
    hasOpenEndedLanguage = segment.description.containsAny([
        "other", "additional", "various", "certain", "selected"
    ])
    
    // Partial evidence: some countries named + open-ended language
    IF explicitCountries.length > 0 AND hasOpenEndedLanguage:
        RETURN TRUE
    
    RETURN FALSE

- China: mentioned
- Japan: mentioned
- "other Asian countries": ambiguous

1. Start with explicit countries: {China, Japan}
2. Add similar countries in same region: {South Korea, Taiwan, Singapore, ...}
3. Apply IndustryDemandProxy weighting

Result: 40% distributed with China/Japan getting direct allocation,
        residual distributed to similar Asian countries

5.1.4 RF-C: Domestic + Foreign Bucket

RF-C Trigger: Two-segment structure with one domestic segment and one ambiguous foreign segment. 
FUNCTION isRF_C_Case(segments, homeCountry):
    // Check for exactly 2 segments
    IF segments.length != 2:
        RETURN FALSE
    
    // Check if one segment is home country
    hasDomestic = FALSE
    hasForeign = FALSE
    
    FOR EACH segment IN segments:
        IF segment.name == homeCountry:
            hasDomestic = TRUE
        ELSE IF segment.name.containsAny(["foreign", "international", "overseas"]):
            hasForeign = TRUE
    
    RETURN hasDomestic AND hasForeign

RF-C Example: Domestic vs. Foreign

Segments:
1. "United States": $120M (60%)
2. "Foreign": $80M (40%)

Analysis:
- Domestic segment: clear (United States)
- Foreign segment: ambiguous (could be any non-US country)

RF-C Universe Construction:
1. Exclude United States (domestic)
2. Include all other countries
3. Apply GDP × SectorPrior weighting

Result: 60% to US (direct), 40% distributed across ~150 foreign countries

5.1.5 RF-D: Multiple Ambiguous Segments

RF-D Trigger: Multiple segments with overlapping or unclear boundaries. 
FUNCTION isRF_D_Case(segments):
    ambiguousSegments = 0
    
    FOR EACH segment IN segments:
        IF isAmbiguous(segment):
            ambiguousSegments++
    
    // RF-D: 2+ ambiguous segments
    RETURN ambiguousSegments >= 2

5.2 Case Selection Decision Tree

5.2.1 Complete RF Decision Logic

FUNCTION selectRFCase(segment, allSegments, evidence, homeCountry):
    // Priority 1: Check RF-A (non-standard region)
    IF isRF_A_Case(segment.name):
        RETURN {
            case: "RF-A",
            reason: "Non-standard region naming",
            universe: constructRF_A_Universe(homeCountry, sector)
        }
    
    // Priority 2: Check RF-C (domestic + foreign)
    IF isRF_C_Case(allSegments, homeCountry):
        RETURN {
            case: "RF-C",
            reason: "Domestic + ambiguous foreign bucket",
            universe: constructRF_C_Universe(homeCountry)
        }
    
    // Priority 3: Check RF-B (partial evidence)
    IF isRF_B_Case(segment, evidence):
        RETURN {
            case: "RF-B",
            reason: "Partial country evidence + incomplete membership",
            universe: constructRF_B_Universe(evidence.explicitCountries, sector)
        }
    
    // Priority 4: Check RF-D (multiple ambiguous)
    IF isRF_D_Case(allSegments):
        RETURN {
            case: "RF-D",
            reason: "Multiple ambiguous segments",
            universe: constructRF_D_Universe(allSegments, sector)
        }
    
    // Default: No RF case applies (use GF instead)
    RETURN {
        case: "NONE",
        reason: "No RF case applies, use GF"
    }

6. Direct + Residual Allocation Routing

6.1 Direct Allocation Methodology

6.1.1 What is Direct Allocation?

Definition: Direct allocation assigns weights to countries that have explicit evidence from structured tables or high-confidence narrative text.

6.1.2 Direct Allocation Algorithm

FUNCTION allocateDirect(evidence, channel):
    directAllocation = {}
    totalAllocated = 0.0
    
    // Step 1: Extract direct evidence from structured tables
    IF evidence.structured EXISTS:
        FOR EACH country, weight IN evidence.structured.countries:
            IF isActualCountry(country):
                directAllocation[country] = weight
                totalAllocated += weight
    
    // Step 2: Add direct evidence from narrative (if high confidence)
    IF evidence.narrative EXISTS:
        FOR EACH mention IN evidence.narrative.countryMentions:
            IF mention.label == "direct" AND mention.confidence > 0.85:
                IF mention.country NOT IN directAllocation:
                    directAllocation[mention.country] = mention.weight
                    totalAllocated += mention.weight
    
    // Step 3: Calculate residual portion
    residualPortion = 1.0 - totalAllocated
    
    RETURN {
        directAllocation: directAllocation,
        totalAllocated: totalAllocated,
        residualPortion: residualPortion
    }

6.1.3 Direct Allocation Rules

Rule Description Priority
R1: Structured First Structured table data always takes precedence Highest
R2: No Overwrite Once a country has direct evidence, don't overwrite High
R3: Confidence Threshold Narrative evidence must have >85% confidence Medium
R4: Sum Validation Total direct allocation must be ≤ 100% Critical

6.2 Residual Allocation Methodology

6.2.1 What is Residual Allocation?

Definition: Residual allocation distributes the remaining weight (after direct allocation) using fallback methods (SSF, RF, or GF).

6.2.2 Residual Allocation Decision Logic

FUNCTION allocateResidual(residualPortion, segment, evidence, sector, homeCountry, channel):
    // No residual to allocate
    IF residualPortion < 0.01: // <1%
        RETURN {}
    
    // Determine fallback type
    fallbackDecision = decideFallback(
        segment.name,
        segment.countries,
        evidence.narrative.countries,
        evidence.structured.explicitCountries,
        sector,
        homeCountry,
        channel,
        residualPortion,
        evidence.countriesWithEvidence,
        evidence.knownZeroCountries,
        evidence.allSegmentCountries
    )
    
    // Apply appropriate fallback
    IF fallbackDecision.type == "segment-specific": // SSF
        residualAllocation = applySSF(
            fallbackDecision.regionCountries,
            residualPortion,
            sector,
            channel
        )
    ELSE IF fallbackDecision.type == "restricted": // RF
        residualAllocation = applyRF(
            fallbackDecision.restrictedSet,
            residualPortion,
            sector,
            channel
        )
    ELSE IF fallbackDecision.type == "global": // GF
        residualAllocation = applyGF(
            residualPortion,
            sector,
            evidence.countriesWithEvidence,
            evidence.knownZeroCountries
        )
    ELSE:
        residualAllocation = {}
    
    RETURN residualAllocation

6.3 Channel-Specific Routing

6.3.1 Revenue Channel Routing

FUNCTION routeRevenueChannel(evidence, sector, homeCountry):
    // Step 1: Direct allocation from revenue tables
    directResult = allocateDirect(evidence, "revenue")
    
    // Step 2: Residual allocation
    IF directResult.residualPortion > 0.01:
        residualAllocation = allocateResidual(
            directResult.residualPortion,
            evidence.segment,
            evidence,
            sector,
            homeCountry,
            "revenue"
        )
        
        // Step 3: Merge direct + residual
        finalAllocation = merge(directResult.directAllocation, residualAllocation)
    ELSE:
        finalAllocation = directResult.directAllocation
    
    // Step 4: Normalize to ensure sum = 1.0
    finalAllocation = normalize(finalAllocation)
    
    RETURN finalAllocation

6.3.2 Supply Chain Channel Routing

FUNCTION routeSupplyChannel(evidence, sector, homeCountry):
    // Step 1: Check for sustainability report (highest priority)
    IF evidence.sustainabilityReport EXISTS:
        RETURN allocateFromSustainabilityReport(evidence.sustainabilityReport)
    
    // Step 2: Check for SEC supplier list
    IF evidence.supplierList EXISTS:
        directResult = allocateDirect(evidence, "supply")
    ELSE:
        // Step 3: Use narrative evidence
        directResult = allocateFromNarrative(evidence.narrative, "supply")
    
    // Step 4: Residual allocation (RF only, never GF for supply)
    IF directResult.residualPortion > 0.01:
        residualAllocation = allocateResidual(
            directResult.residualPortion,
            evidence.segment,
            evidence,
            sector,
            homeCountry,
            "supply"
        )
        finalAllocation = merge(directResult.directAllocation, residualAllocation)
    ELSE:
        finalAllocation = directResult.directAllocation
    
    RETURN normalize(finalAllocation)

6.3.3 Physical Assets Channel Routing

FUNCTION routeAssetsChannel(evidence, sector, homeCountry):
    // Priority 1: Exhibit 21 subsidiaries
    IF evidence.exhibit21 EXISTS:
        RETURN allocateFromExhibit21(evidence.exhibit21)
    
    // Priority 2: PP&E geographic table
    IF evidence.ppeTable EXISTS:
        directResult = allocateDirect(evidence, "assets")
    // Priority 3: Sustainability report facilities
    ELSE IF evidence.sustainabilityReport EXISTS:
        RETURN allocateFromSustainabilityFacilities(evidence.sustainabilityReport)
    // Priority 4: Facility locations (narrative)
    ELSE IF evidence.facilityLocations EXISTS:
        directResult = allocateFromFacilities(evidence.facilityLocations)
    ELSE:
        // Priority 5: Fallback
        RETURN applyAssetsFallback(sector, homeCountry)
    
    // Residual allocation if needed
    IF directResult.residualPortion > 0.01:
        residualAllocation = allocateResidual(
            directResult.residualPortion,
            evidence.segment,
            evidence,
            sector,
            homeCountry,
            "assets"
        )
        finalAllocation = merge(directResult.directAllocation, residualAllocation)
    ELSE:
        finalAllocation = directResult.directAllocation
    
    RETURN normalize(finalAllocation)

6.3.4 Financial Channel Routing

FUNCTION routeFinancialChannel(evidence, sector, homeCountry):
    // Step 1: Check for debt securities table
    IF evidence.debtTable EXISTS:
        directResult = allocateDirect(evidence, "financial")
    // Step 2: Check for treasury centers
    ELSE IF evidence.treasuryCenters EXISTS:
        directResult = allocateFromTreasuryCenters(evidence.treasuryCenters)
    ELSE:
        // Step 3: Global fallback (financial always uses GF, never SSF/RF)
        RETURN applyFinancialGlobalFallback(sector, homeCountry)
    
    // Residual allocation (GF only)
    IF directResult.residualPortion > 0.01:
        residualAllocation = applyGF(
            directResult.residualPortion,
            sector,
            evidence.countriesWithEvidence,
            evidence.knownZeroCountries
        )
        finalAllocation = merge(directResult.directAllocation, residualAllocation)
    ELSE:
        finalAllocation = directResult.directAllocation
    
    RETURN normalize(finalAllocation)

7. Appendix & References

7.1 Key Formulas Summary

Blended Weight Formula:

Wblended(c) = α·Wr(c) + β·Ws(c) + γ·Wp(c) + δ·Wf(c)

where α=0.40, β=0.35, γ=0.15, δ=0.10 (default)
SSF Weight Formula:

WSSF(c) = [IndustryDemandProxy(c) / Σ IndustryDemandProxy(c)] × Wresidual

where c ∈ R (countries in defined region R)
RF Weight Formula:

WRF(c) = [GDP(c) × SectorPlausibility(c) / Σ GDP(c) × SectorPlausibility(c)] × Wresidual

where c ∈ P (restricted plausible set P)
GF Weight Formula:

WGF(c) = [GDP(c) × SectorPrior(c) / Σ GDP(c) × SectorPrior(c)] × Wresidual

where c ∈ Global (all countries)
