# CO-GRI v3.4 Fallback Logic Upgrade Plan

## PHASE 1-2: ANALYSIS & ARCHITECTURE DESIGN

### Current System Analysis (v3.3)

#### Existing Components:
1. **fallbackLogic.ts** - Three-tier fallback system (SSF/RF/GF)
2. **geographicExposureService.ts** - Main service with SEC integration
3. **restrictedFallback.ts** - RF implementation
4. **channelSpecificFallback.ts** - Channel-specific formulas
5. **structuredDataIntegrator.ts** - SEC filing data integration

#### Current Evidence Hierarchy (v3.3):
1. Company-Specific Overrides (highest priority)
2. SEC Filing Structured Data
3. Narrative Evidence
4. Three-Tier Fallback (SSF → RF → GF)

### NEW v3.4 ARCHITECTURE DESIGN

#### Enhanced 4-Tier Evidence Hierarchy:
1. **Structured Evidence** (Highest Priority)
   - SEC filing tables with explicit country/region data
   - Company-specific overrides
   - Regulatory filings with geographic breakdowns

2. **Narrative Evidence** (Second Priority)
   - Text mentions of countries/regions in filings
   - Management discussion sections
   - Risk factor disclosures

3. **Supplementary Evidence** (Third Priority)
   - Industry reports and third-party data
   - Trade flow data from UN Comtrade
   - Supply chain intelligence

4. **Fallback Evidence** (Lowest Priority)
   - Mathematical fallback formulas
   - Sector-specific demand proxies
   - GDP-weighted global distributions

#### Evidence Sufficiency Framework:
- **Sufficient**: >90% coverage with high-confidence evidence
- **Partial**: 50-90% coverage with mixed confidence
- **Insufficient**: <50% coverage, requires fallback

#### Advanced Caching Architecture:
1. **Document Cache**
   - Raw filing data with metadata
   - Parsing results with confidence scores
   - Evidence attribution tracking

2. **Evidence-Confirmed Cache**
   - Validated geographic exposures
   - Supersession rules for newer data
   - Quality metrics and timestamps

#### Jurisdiction-Aware Processing:
- **U.S. Listed Companies**: Full SEC Edgar integration
- **Non-U.S. Listed**: International regulatory APIs
- **Unlisted Companies**: Alternative data sources
- **Cross-Border Entities**: Multi-jurisdiction reconciliation
